# fan-out並列

## 手順

1. 作業を独立した単位に分割する（worker 間に依存がないことを確認する）
2. 各 worker への指示を作る：タスク＋出力先 `work/<案件名>/<単位名>.md`＋報告形式（要約＋パス）
3. 全 worker を**1つのメッセージにまとめて**起動する（並列実行になる）
4. 全報告を集約し、担当割りの隙間（抜け漏れ）がないか点検してから上へ報告する

## 注意

- 出力先は worker 毎に必ず分ける（書き込み衝突の防止）
- 分割数はコストと相談する。10を超えるなら lead を挟んで束ねる
