# 競作

## 手順

1. お題と評価基準を先に決める（何をもって「良い」とするかを定義してから始める）
2. 複数の worker に**異なるアプローチ・観点**を割り当て、1メッセージで並列起動する（出力先は `work/<案件名>/案A.md` 等で分離）
3. 全案が出揃ったら評価基準で比較する。必要なら検査役 worker に採点させる
4. 最良案を軸に、次点の良い要素を取り込んで確定する

## 注意

- コストは案の数に比例する。通常は2〜3案にとどめる
- アプローチの指定を変えないと似た案が複数返ってくる。観点の割り当てが要
