# パイプライン

## 手順

1. 工程を定義し、各工程の成果物パスを決める（`work/<案件名>/01-調査.md` のように順序を付ける）
2. 工程1の worker を起動する（出力先を指示に含める）
3. 完了報告を検品し、工程2の worker に**前工程の成果物パスを指示に含めて**起動する
4. 最終工程まで繰り返し、全体を要約して上へ報告する

## 注意

- 工程間の受け渡しは必ずファイルパスで行う（報告文の又聞きで内容を伝えない）
- 工程の途中で品質が怪しければ、次に進めず前工程をやり直させる
