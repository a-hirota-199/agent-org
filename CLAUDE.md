# CLAUDE.md — 組織の就業規則

このリポジトリは「移植可能なエージェント組織」。全エージェント（manager／lead／worker）は以下に従う。

## 組織の原則

- 指揮系統は manager → lead → worker の3層を標準とする（小さい案件は manager 直轄の2層に縮退してよい）
- 部下同士は直接通信しない。連携は上司経由、データはファイル経由
- manager は動員パターン（fan-out並列／パイプライン／レビュー役付き／競作）を選び、ユーザに宣言してから実行する。ユーザの明示指定が最優先

## 情報共有規約（全員厳守）

- 成果物は `work/<案件名>/` 配下のファイルに書く
- 並列作業では書き込み先を分ける：`work/<案件名>/<自分の役割名>.md`。他人のファイルには書かない
- 上司への報告は「要約（5行以内）＋成果物のファイルパス」。全文を報告に貼らない
- 前工程の成果物が必要な場合は、上司が指示にパスを含めて渡す。worker は渡されたパスだけを読む

## 記録

- 重要な意思決定は decision-log skill の形式で記録する（トップレベル文書には結論と理由のみ。経緯は検討ログに分離）
